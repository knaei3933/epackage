## 🔴 CSPエラー修正（最重要）

### 問題の概要

Google Search Consoleで以下のCSP違反エラーが報告されました：

```
Loading the stylesheet 'https://www.package-lab.com/_next/static/css/...'
violates the following Content Security Policy directive: "style-src 'self' 'unsafe-inline'"

Loading the script 'https://www.package-lab.com/_next/static/chunks/...'
violates the following Content Security Policy directive: "script-src 'self' 'unsafe-inline' 'unsafe-eval' https://js.sendgrid.com"
```

### 根本原因

**ファイル**: `src/middleware.ts`

**問題**: パスチェックの不一致
- Line 335: `/_next`（スラッシュなし）
- Line 967: `/_next/`（スラッシュあり）

この不一致により、Next.jsの静的ファイル（`/_next/static/`）に対してもCSPヘッダーが適用されていました。

### 修正内容

**コミット**: 9af3456

1. **Line 339**: 静的パスに対して `addSecurityHeaders()` をスキップ
```typescript
if (isStaticPath) {
  // Static assets don't need security headers - prevents CSP on static chunks
  return NextResponse.next();
}
```

2. **Line 738**: 同様に静的パスではセキュリティヘッダーをスキップ

3. **Line 742-749**: 重複した静的パスチェックを削除

4. **Line 967**: パスチェックの修正
```typescript
// 修正前: if (!pathname || !pathname.startsWith('/_next/'))
// 修正後: if (!pathname || !pathname.startsWith('/_next'))
if (!pathname || !pathname.startsWith('/_next')) {
```

### 検証結果

**本番環境でのcurl検証**:
```bash
curl -I https://www.package-lab.com/_next/static/chunks/...
```

**結果**: CSPヘッダーが正常に削除されていることを確認

### middleware.tsの重要な変更点

// Line 335-341: 静的パスチェック
const isStaticPath = STATIC_PATHS.some(p => pathname.startsWith(p)) ||
  pathname.match(/\.(ico|png|jpg|jpeg|gif|svg|css|js|woff|woff2)$/);

if (isStaticPath) {
  // Static assets don't need security headers - prevents CSP on static chunks

}
