## 1. GTMプレビューモード検証

### 1.1 検証準備

**必要なツール:**
- Google Tag Managerアカウントアクセス
- Google Chromeブラウザ（推奨）
- Tag Assistant Companion（Chrome拡張機能）

**前提条件:**
- [x] GTMコンテナID: `GTM-T4PL5XMC`
- [x] GA4測定ID: `G-VBCB77P21T`
- [x] GTMスニペットがサイトに実装済み

### 1.2 プレビューモード起動手順

**ステップ1: GTMでプレビューモード開始**

```
1. [GTMコンソール](https://tagmanager.google.com/)にアクセス
2. ワークスペース「GTM-T4PL5XMC」を選択
3. 画面右上「プレビュー」ボタンをクリック
4. 「接続」ボタンをクリック
5. 新しいタブでサイトURL（https://package-lab.com）を開く
```

**ステップ2: Tag Assistantの確認**

```
1. 画面左側に「Tag Assistant」サイドバーが表示されることを確認
2. 「Connected to GTM」ステータスを確認
3. コンテナバージョンが最新であることを確認
```

### 1.3 各コンバージョンポイントの検証

#### 1.3.1 見積もり完了（quote_complete）検証

**検証手順:**

```
1. 見積もりシミュレーターページへ移動
   URL: /quote-simulator

2. 見積もり条件を入力:
   - 製品カテゴリ: 選択

   - サイズ: 選択
   - 材質: 選択
   - その他オプション: 適宜選択

3. 「見積もりを完了」ボタンをクリック

4. Tag Assistantで以下を確認:
   - quote_complete イベント発火
   - dataLayerパラメータ確認:
     * currency: "JPY"
     * value: [見積もり金額]
     * quote_id: [見積もりID]

**期待されるTag Assistant出力:**

```

├── currency: JPY
├── value: 150000
├── quote_id: QUOTE-12345
└── event: quote_complete

## 5. テスト運用スケジュール（4週間）

### 5.1 全体スケジュール概要

```
期間: 開始日から4週間
予算: 月5万円程度
配信プラットフォーム: Google Ads 80%, Yahoo! 20%
目的: トラッキング検証、初期データ収集、最適化実施
```

### 5.2 週別スケジュール

#### 第1週目: 低予算テスト配信開始

**目的:** トラッキング検証と初期データ収集

**キャンペーン設定:**

| キャンペーン | 日予算 | 週予算 | 入札戦略 | 目標 |
|-------------|--------|--------|----------|------|
| ブランドキーワード | 500円 | 3,500円 | 目標CPA 3,000円 | トラッキング検証 |
| 一般キーワード | 1,500円 | 10,500円 | 目標CPA 8,000円 | トラッキング検証 |
| ロングテール | 500円 | 3,500円 | 目標CPA 3,000円 | トラッキング検証 |
| **週合計** | **2,500円** | **17,500円** | - | - |

**デイリータスク:**

| 日 | タスク | 確認項目 |
|----|--------|----------|
| 月日 | 配信開始 | インプレッション発生 |
| 火 | トラッキング確認 | コンバージョン記録 |
| 水 | 検索語確認 | 除外KW候補抽出 |
| 木 | 品質スコア確認 | 低品質KW調整 |
| 金 | 週次レポート作成 | CPA/CVR確認 |
| 土 | データ分析 | 週末傾向確認 |
| 日 | 次週計画 | 調整事項決定 |

**検証重点項目:**
- [ ] 全コンバージョンイベントが正しく記録される
- [ ] Google Adsでコンバージョンが表示される
- [ ] Yahoo!広告でコンバージョンが表示される
- [ ] CPAが目標範囲内である
- [ ] 検索語レポートで不要KWを特定

#### 第2週目: パフォーマンス分析

**目的:** データ分析と初期調整

**キャンペーン調整:**

| アクション | 内容 | 実施基準 |
|-----------|------|----------|
| 除外KW追加 | 不要KWを除外 | CTR 1%未満、CV 0 |

## 8. トラブルシューティング

### 8.1 トラッキング関連の問題

#### 問題: コンバージョンが記録されない

**症状:**
- GTMプレビューではイベント発火している
- GA4デバッグビューではイベント表示される
- Google Adsでコンバージョンが記録されない

**原因:**
1. GA4とGoogle Adsの連携が完了していない
2. コンバージョンインポート設定が不備
3. データの反映遅延（最大24時間）

**対処手順:**

```
1. 連携確認
   Google Ads > ツールと設定 > リンク済みアカウント
   GA4連携が「有効」であることを確認

2. コンバージョン設定確認
   ツールと設定 > コンバージョン
   各コンバージョンのステータスが「有効」であることを確認

3. インポート設定確認
   コンバージョン詳細 > 設定
   GA4イベントが正しくインポートされていることを確認

4. 反映待ち
   コンバージョンは最大24時間かかるため、翌日まで待つ

5. それでも記録されない場合
   - Googleサポートに連絡
   - GTM/GA4設定を再確認
```

#### 問題: dataLayerパラメータが正しく渡されない

**症状:**
- イベントは発火する
- パラメータ値がnullやundefinedになる
- 動的価値が記録されない

**原因:**
1. タイミングの問題（データ取得前にイベント発火）
2. 変数名の不一致
3. 値が正しく取得できていない

**対処手順:**

```
1. 実装コード確認
   src/lib/analytics/dataLayer.ts
   関数呼び出し時に正しい値が渡されているか確認

2. GTM変数確認
   GTM > 変数 > データレイヤー変数
   変数名が一致しているか確認

3. タイミング確認
   イベント発火前に値が設定されているか確認
   必要であればsetTimeoutで遅延させる

4. デバッグ
   console.logで値を確認
   ブラウザ開発者ツールでdataLayerを確認
```

#### 問題: Yahoo!タグが発火しない

**症状:**
- Yahoo!タグヘルパーでタグが検知されない
- 共通タグが発火しない

**原因:**
1. Yahoo!タグが実装されていない
2. サイトIDが間違っている
3. 広告ブロッカーが有効

**対処手順:**

```
1. タグ実装確認
   サイトのソースコードでYahoo!タグを確認
   正しい位置（<head>内）に実装されているか確認

2. サイトID確認
   Yahoo!広告管理ツールで正しいサイトIDを確認
   タグ内のサイトIDが正しいか確認

3. 広告ブロッカー無効化

4. Yahoo!タグヘルパー再起動
   ブラウザ拡張機能を再起動

   a. 競合が要因の場合
      - 入札単価調整
      - ターゲット広告文改善

   b. LPが要因の場合
      - LP改善実施
      - 別LPテスト

   c. ターゲティングが要因の場合
      - ターゲット調整
      - 除外設定追加

4. モニタリング強化
   - デイリーでパフォーマンス確認
   - 早期異常検知体制構築
```

#### 問題: インプレッションが激減した

**症状:**
- インプレッションが前日比で50%以上減少
- クリック数も減少

**原因:**
1. 入札単価が低すぎる
2. 予算不足
3. 配信停止になった
4. キーワードの品質スコア低下

**対処手順:**

```
1. 即時確認
   a. 入札単価確認
      - 入札単価が競合より低すぎないか
      - 入札上限設定の確認

   b. 予算確認
      - 日予算を使い切っていないか
      - 月予算の達成状況確認

   c. 配信ステータス確認
      - キャンペーンが「有効」であるか
      - キーワードが「有効」であるか

   d. 品質スコア確認
      - 急激に低下していないか

2. 対応実施
   a. 入札単価の場合
      - 入札単価を+20%程度上げる
      - 入札制限を見直す

   b. 予算の場合
      - 予算を増額する
      - 予算配分を再検討

   c. 品質スコアの場合
      - 広告文改善
      - ランディングページ改善
      - キーワードの関連性向上

3. モニタリング
   - 毎時間インプレッション確認
   - 改善効果の測定
```

#### 問題: クリック数が多いがコンバージョンが少ない

**症状:**
- CTRは高い
- CVRが低い（1%未満）
- CPAが高い

**原因:**
1. ランディングページの問題
2. キーワードとLPの不一致
3. ターゲットユーザーの不一致
4. コンバージョン導線の問題

**対処手順:**

```
1. ランディングページ分析
   a. Google Analyticsで確認
      - 直帰率
      - 平均セッション時間
      - ページ別離脱率

   b. ユーザビリティ確認
      - モバイル表示の確認
      - フォームの使いやすさ
      - ページ速度（Core Web Vitals）

2. キーワードとLPの一致確認
   a. 広告文とLPの一致
      - 広告文の約束をLPで果たしているか

   b. キーワードとLPの一致
      - キーワードに関連する情報がLPにあるか

3. ターゲットユーザー確認
   a. 検索意図の確認
      - 情報収集なのか購入意図があるのか

   b. オーディエンス確認
      - ターゲット層に合っているか

4. コンバージョン導線確認
   a. フォームまでの導線
      - クリック数が多すぎないか
      - フォームが見つけやすいか

   b. フォームの使いやすさ
      - 入力項目が多すぎないか
      - エラーが分かりやすいか

5. 改善実施
   a. A/Bテスト実施
      - 複数パターンのLPをテスト

   b. 広告文調整
      - より具体的な広告文に変更

   c. ターゲティング調整
      - より購入意図の強い層に絞る
```

### 8.3 技術的な問題

#### 問題: ランディングページが表示されない

**症状:**
- 広告クリック後にLPが表示されない
- 404エラーが発生する
- ページ読み込みが遅い

**原因:**
1. サーバーダウン
2. DNS設定の問題
3. URLの間違い
4. サーバーの負荷

**対処手順:**

```
1. 即時確認
   a. サイトステータス確認
      - サーバーが稼働しているか
      - 他のページも表示されるか

   b. URL確認
      - 広告のリンク先URLが正しいか
      - パラメータが正しく設定されているか

2. 対応実施
   a. サーバー問題の場合
      - サーバー管理者に連絡
      - バックアップサーバーに切り替え

   b. URL問題の場合
      - URLを修正
      - 広告を一時停止して修正

3. 広告対応
   - 即時: 広告を一時停止
   - 復旧後: 広告を再開
   - 損失記録: 影響を記録
```

#### 問題: フォーム送信できない

**症状:**
- フォーム送信時にエラーが発生する
- 送信ボタンが反応しない
- バリデーションエラーが表示される

**原因:**
1. フロントエンドの問題
2. APIエラー
3. バリデーションの問題
4. サーバーの問題

**対処手順:**

```
1. エラー確認
   a. ブラウザコンソール確認
      - JavaScriptエラーの有無
      - APIエラーの有無

   b. ネットワーク確認
      - APIリクエストが成功しているか
      - ステータスコードの確認

2. 原因特定
   a. フロントエンドの場合
      - JavaScriptのバグ特定
      - フォームバリデーション確認

   b. APIの場合
      - サーバーログ確認
      - APIエンドポイント確認

3. 対応実施
   a. 緊急対応
      - 問題の切り分け
      - 最小限の修正で復旧

   b. 根本対応
      - バグ修正
      - テスト実施

4. 広告対応
   - 問題解決まで広告を一時停止
   - 復旧後広告を再開
```
