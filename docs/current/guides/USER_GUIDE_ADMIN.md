# B2B 시스템 관리자 사용자 가이드

## 시스템 개요

Epackage Lab B2B 시스템은 10단계 주문 프로세스를 관리하는 종합 플랫폼입니다.

### 10단계 프로세스
1. **견적 (Quotation)** - 고객 견적 요청 접수
2. **데이터 입고 (Data Received)** - 디자인 데이터 수령
3. **작업표준서 (Work Order)** - 제조 사양서 작성
4. **계약서 송부 (Contract Sent)** - 계약서 전송
5. **계약서 서명 (Contract Signed)** - 양측 전자서명
6. **생산 (Production)** - 제조 공정 진행
7. **입고 (Stock In)** - 제품 입고 및 품질검사
8. **출하 (Shipped)** - 배송 시작
9. **배송 완료 (Delivered)** - 고객 수령 완료

---

## 1. 대시보드 (Dashboard)

### 접속 경로
```
/admin/dashboard
```

### 주요 기능

#### 통계 카드
- **미처리 견적**: 승인 대기 중인 견적 수
- **진행 중 주문**: 현재 생산/처리 중인 주문
- **금일 완료**: 오늘 완료된 주문 수
- **월간 매출**: 이번 달 총 매출액

#### 최근 활동
- 최근 견적 요청
- 최근 주문
- 서명 대기 계약서
- 생산 중 주문

---

## 2. 견적 관리 (Quotation Management)

### 견적 목록 보기
```
/admin/quotations
```

### 견적 승인 절차

1. **견적 상세 보기**
   - 목록에서 견적 클릭
   - 고객 정보 확인
   - 요청 항목 확인

2. **가격 계산**
   - 단가 입력
   - 수량 확인
   - 합계 계산 (자동)

3. **견적 승인**
   - `견적 승인` 버튼 클릭
   - 유효기간 설정 (기본 30일)
   - 메모 추가 (선택)

4. **PDF 생성**
   - 승인 후 자동 생성
   - 다운로드 가능

---

## 3. 작업표준서 관리 (Work Order)

### 작업표준서 생성
```
/admin/work-orders/new
```

### 필수 항목

| 항목 | 설명 | 필수 여부 |
|------|------|----------|
| 주문 선택 | 견적이 승인된 주문 | 필수 |
| 인쇄 방식 | 오프셋, 디지털 등 | 필수 |
| 용지 사양 | 종류, 평량 | 필수 |
| 크기 | 가로 x 세로 x 높이 | 필수 |
| 코팅 | 코팅 여부 및 종류 | 선택 |
| 후공정 | 박금, 형압, 부분 UV 등 | 선택 |
| 특이사항 | 주의 사항 | 선택 |

### 생성 후 작업
- PDF 다운로드
- 고객에게 공유
- 생산 팀에 전달

---

## 4. 계약서 관리 (Contract Management)

### 계약서 생성
```
/admin/contracts/new
```

### 계약서 생성 절차

1. **주문 선택**
   - 계약을 생성할 주문 선택
   - 작업표준서가 완료된 주문만 표시

2. **계약 조건 입력**
   - 결제 조건: 예) "검수 후 30일 지급"
   - 인도 조건: 예) "공장 인도"
   - 계약 기간 설정

3. **계약서 생성**
   - `계약서 생성` 버튼 클릭
   - PDF 자동 생성

4. **관리자 서명**
   - 계약서 미리보기
   - 약관 동의 체크
   - 서명란에 서명
   - `서명 완료` 클릭

### 서명 상태 확인
- **대기 (Pending)**: 고객 서명 대기
- **부분 완료**: 관리자 서명 완료
- **완료 (Completed)**: 양측 서명 완료

---

## 5. 생산 관리 (Production Management)

### 생산 대시보드
```
/admin/production
```

### 생산 시작

1. **주문 선택**
   - 계약이 완료된 주문 선택

2. **담당자 지정**
   - 생산 담당자 선택
   - 예상 완료일 설정

3. **생산 시작**
   - `생산 시작` 버튼 클릭
   - 상태가 `PRODUCTION`으로 변경

### 생산 공정 업데이트

#### 9단계 생산 공정
1. **prepress** (제판)
2. **printing** (인쇄)
3. **die_cutting** (다이컷)
4. **folding** (접지)
5. **gluing** (접착)
6. **inspection** (검사)
7. **packaging** (포장)
8. **qc** (품질검사)
9. **completed** (완료)

#### 업데이트 절차
1. 현재 공정 선택
2. 진척률 입력 (0-100%)
3. 작업 메모 입력
4. 작업 사진 업로드 (선택)
5. `업데이트` 클릭

### 진척률 계산
- 각 공정 완료 시 자동 계산
- 9단계 기준: `(완료 단계 / 9) * 100%`

---

## 6. 입고 처리 (Stock In)

### 입고 페이지
```
/admin/stock-in
```

### 입고 절차

1. **주문 선택**
   - 생산 완료된 주문 선택

2. **입고 정보 입력**
   - 입고 수량
   - 창고 위치 (예: A-01-03)
     - A: 구역
     - 01: 선반
     - 03: 칸

3. **품질 검사 결과**
   - **합격 (PASSED)**: 정상 입고
   - **불합 (FAILED)**: 불량품 처리
   - **재작업 (REWORK)**: 재작업 필요

4. **검사 메모**
   - 검사 결과 메모
   - 불량 사유 등

5. **입고 사진**
   - 입고 제품 사진
   - (선택 사항)

6. **입고 완료**
   - `입고 완료` 클릭
   - 상태가 `STOCK_IN`으로 변경

---

## 7. 출하 처리 (Shipment)

### 출하 페이지
```
/admin/shipments
```

### 출하 절차

1. **주문 선택**
   - 입고 완료된 주문 선택

2. **송장 번호 입력**
   - 형식: INV-YYYY-NNNN
   - 예: INV-2025-0001

3. **배송업체 선택**
   - 야마토 운송 (ヤマト運輸)
   - 사가와 급속 (佐川急便)
   - 우편 (ゆうパック)
   - 세이노 운수 (西濃運輸)
   - 후쿠야마 운송 (福山通運)
   - 기타

4. **운송장 번호**
   - 배송업체 운송장 번호

5. **추적 URL**
   - 배송 추적 페이지 URL
   - (선택 사항)

6. **출하일**
   - 실제 출하 날짜
   - 기본: 오늘

7. **출하 완료**
   - `출하 완료` 클릭
   - 고객에게 알림 자동 발송

---

## 8. 주문 추적 (Order Tracking)

### 추적 페이지
```
/admin/orders/[id]
```

### 추적 정보
- 현재 단계 표시
- 진척률 그래프
- 완료 단계 (체크 표시)
- 현재 단계 (파랑색 표시)
- 미완료 단계 (회색)

### 상태 변경 이력
- 모든 상태 변경 기록
- 변경 일시
- 변경 담당자
- 변경 사유

## 11. 샘플 요청 관리 (Sample Requests)

### 샘플 목록
```
/admin/samples
```

### 샘플 처리 절차

1. **샘플 요청 확인**
   - 요청 제품
   - 수량
   - 배송 주소

2. **샘플 발송**
   - 상태를 `SHIPPED`로 변경

3. **샘플 도착**
   - 상태를 `DELIVERED`로 변경
   - 고객 피드백 요청

## 주의 사항

### 필수 확인 사항
1. **작업표준서**: 계약 전에 반드시 생성
2. **계약서**: 생산 시작 전에 양측 서명 완료
3. **생산 공정**: 각 단계마다 업데이트
4. **입고**: 품질검사 후 입고 처리
5. **출하**: 운송장 번호 필수 입력

### 오류 처리
- **데이터 오류**: 시스템 관리자에게 문의
- **PDF 생성 오류**: 브라우저 새로고침 후 재시도
- **서명 오류**: 페이지를 새로고침하고 다시 시도
