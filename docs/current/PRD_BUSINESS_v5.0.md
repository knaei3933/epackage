## 2. 12ステージB2Bワークフロー（PRD v2.1より継承）

### 2.1 ワークフロー全体図

```
Stage 1: 見積シミュレーション (公開)
Stage 2: 顧客情報入力 (登録なし)
Stage 3: 見積発行・会員登録誘導
Stage 4: 会員登録・承認待ち
Stage 5: 入稿データダウンロード
Stage 6: データ入稿（顧客）
Stage 7: 韓国デザイナー送付・修正依頼
Stage 8: 修正データ完了通知
Stage 9: 顧客承認（修正依頼ループ）
Stage 10: 計算発行・入金待ち
Stage 11: 製造開始（入金確認）
Stage 12: 発注書送付・製造
Stage 13: 納品書送付・完了
```

### 2.2 ステージ詳細

#### Stage 1-3: 見積・会員獲得
- **アクター**: 見込み顧客
- **機能**:
  - AI見積シミュレーター
  - 即座価格提示（¥150,000〜¥500,000）
  - 会員登録誘導

#### Stage 4: 承認プロセス
- **アクター**: 管理者
- **機能**:
  - 企業情報確認
  - 承認/拒否
  - アカウント有効化

#### Stage 5-8: データ入稿・デザイン
- **アクター**: 顧客 → 韓国パートナー → 顧客
- **機能**:
  - デザインデータアップロード
  - AI自動抽出（製品仕様）
  - 韓国パートナー送付（メール）
  - 修正データ返却

#### Stage 9: 顧客承認ループ
- **アクター**: 顧客
- **機能**:
  - プレビュー確認（JPG）
  - 承認または修正依頼
  - Stage 6-8 ループ

#### Stage 10: 計算・入金
- **アクター**: 顧客 → 管理者
- **機能**:
  - 計算書発行（銀行振込のみ）
  - 入金確認
  - 入金完了通知

#### Stage 11-13: 製造・納品
- **アクター**: 管理者 → 韓国パートナー → 顧客
- **機能**:
  - 製造開始（入金+データ承認条件）
  - 発注書送付（韓国→日本）
  - 配送追跡
  - 納品書自動送付
  - 3ヶ月後アーカイブ
