## 1. 概要

本ドキュメントは、見積もりシステムにおける **①フィルム幅計算式、②マチ（gusset）、③シール幅（sealWidth）** の現状ロジックを整理し、今回の要件「**Excel表に基づく後加工シール幅の適用**」を実装するための参照資料である。

### 今回の要件（ユーザー指示）

> 스탠드 파우치, 스파우트파우치와 같이 밑지가 있는 경우 후가공 옵션에 있는 シール幅을 맘대로 정할 수 없고, 각 파우치 폭마다 있는 シール幅이 정해져 있어서 엑셀 표의 내용이 적용되어야 한다.

- 底マチ（gusset）ありパウチでは、シール幅は**パウチ幅×マチに応じて決まる固定の選択肢**であり、ユーザーが自由に選べるべきではない
- Excel 表（D24:J101）のデータを適用する

### 確定した適用方針（deep-interview 7ラウンドで合意）

| 項目 | 方針 |
|------|------|
| 適用範囲 | **UI選択肢の制約のみ**。価格計算（フィルム幅のハードコード定数）は現状維持 |
| 複数パターン | 該当するシール幅候補を**すべて選択肢として提示**しユーザーが選ぶ（1つなら自動決定） |
| J列の2値 `A*B` | **保有するシール幅ツール（金型）の組み合わせ**。`10*7.5`＝10mm工具1つ＋7.5mm工具1つ保有＝両方対応可能 |
| 選択肢生成 | 幅+マチ → 該当レコード群の J列を `{A,B}` に展開 → 全マージ → 重複排除 |
| 格納 | 選ばれた1値を `quoteState.sealWidth`（単一文字列、例 `'5mm'`）に格納 |
| 非対象（従来3択維持） | 底マチなしパウチ（平袋・ロール・合掌）、Excelにないサイズ |

## 2. 現状の計算式（フィルム幅）

> ⚠️ 今回は**対象外**（Round 1 で「UI選択肢制約のみ、価格計算は現状維持」と合意）。参照用。

### 2.1 `src/lib/pouch-cost-calculator.ts`（デジタル印刷）

`calculateSingleColumnFilmWidth`（L885-909）の switch 文。末尾の定数 `+35/+22/+32/+41` が実質的に**シール代（固定値）**として埋め込まれている。

| パウチタイプ | 1列式 | 2列式 |
|--------------|-------|-------|
| `stand_up` / `zipper_stand` / `spout` / `spout_pouch`（マチあり） | `(H×2)+(G×2)+35` | `(H×4)+(G×2)+40` |
| `t_shape`（合掌袋） | `(W×2)+22` | `(W×4)+64` |
| `box` / `m_shape`（ガゼット） | `(G+W)×2+32` | `(G+W)×4+84` |
| `flat_3_side`（平袋三封） | `(H×2)+41` | `(H×4)+71` |

- `H`=高さ、`G`=マチ（=`quoteState.depth`）、`W`=幅
- 末尾定数は計算式の変数ではなく**リテラル**

### 2.2 `src/lib/gravure-material-width.ts`（グラビア印刷）

L37-62 の同名関数。グラビア仕様§3 準拠。デジタル側と**同じ定数値**（`+41/+35/+22/+32`）だが、こちらは1列専用（2列式なし）。L90-91 コメントに「公式の端数定数が既に10mm系の実務値を反映済み」と明記。

### 2.3 注意点

- デジタル/グラビアで定数が**重複定義**。片方だけ変えると不整合（今回は触らないので問題なし）
- 材料費計算時に別途 `+10mm`（最終熱シール層、gravure-material-width.ts:93）が加算される

### 5.2 J列の意味（ユーザー確認済み・Round 4）

> `7.5*7.5` 이건 도구를 두개 가지고 있다는 의미입니다. 따라서 `10*7.5`는 10짜리 하나 7.5 하나 가지고 있어서 둘다 대응 가능하다는 의미입니다.

J列 `A*B` = **保有するシール幅ツール（金型）の組み合わせ**:

| J列 | 意味 | 展開選択肢 |
|-----|------|------------|
| `7.5*7.5` | 7.5mm工具を2つ保有 | `{7.5}` |
| `5*5` | 5mm工具を2つ保有 | `{5}` |
| `10*10` | 10mm工具を2つ保有 | `{10}` |
| `10*7.5` | 10mm工具1つ＋7.5mm工具1つ保有 | `{10, 7.5}` |
| `7.5*6` | 7.5mm工具1つ＋6mm工具1つ保有 | `{7.5, 6}` |
| `10*7.5` | 同上 | `{10, 7.5}` |

### 5.3 選択肢生成ロジック

```
幅(W) + マチ(G)
  ↓ Excel該当レコード群を検索（width==W AND gusset==G）
  ↓ 各レコードの J列 "A*B" を {A, B} に展開
  ↓ 全レコード分をマージ
  ↓ 重複排除（Set）
  ↓ 降順ソート（10, 7.5, 5 等）
選択肢セット
```

**例**:

| 幅+マチ | 該当レコード | 展開 | 選択肢セット |
|---------|--------------|------|--------------|
| 130 + 40 | R40 `5*5`, R41 `10*10` | `{5}`, `{10}` | `{5, 10}` |
| 150 + 45 | R53 `5*5`, R54 `10*7.5` | `{5}`, `{10, 7.5}` | `{5, 7.5, 10}` |
| 200 + 43 | R74 `5*5`, R75 `10*10` | `{5}`, `{10}` | `{5, 10}` |
| 80 + 25 | R24 `7.5*7.5` | `{7.5}` | `{7.5}`（自動決定） |

### 5.4 適用条件

- **対象**: 底マチありパウチ（`stand_up`, `zipper_stand`, `spout_pouch`, `box`, `m_shape` 等、`depth > 0`）
- **非対象（従来3択 `5/7.5/10mm` 維持）**:
  - 底マチなしパウチ（`flat_3_side`, `roll_film`, `lap_seal`）
  - Excel にない幅+マチ組み合わせ（不在サイズ）

### 5.5 データ構造（実装案）

Excel を実行時に読むのは非現実的（Zドライブ・Webアプリ）ため、**Excel D24:J101 を TypeScript 定数テーブルに変換**して新設 `src/lib/sealing-data.ts` に持たせる。

```typescript
// src/lib/sealing-data.ts（新設）
// 元データ: 이패키지 가공.xlsx / 가공1호기 / D24:J101

// 生レコード（1行=1工具保有パターン）
export interface SealingRecord {
  width: number       // パウチ幅 mm
  gusset: number      // マチ mm
  sealPair: [number, number]  // J列 A*B
}

export const SEALING_RECORDS: SealingRecord[] = [
  { width: 80,  gusset: 25, sealPair: [7.5, 7.5] },
  { width: 90,  gusset: 25, sealPair: [7.5, 6] },
  // ... D24:J101 全78レコード
];

// 幅+マチ → 選択肢セット生成
export function getAvailableSealWidths(width: number, gusset: number): number[] {
  const set = new Set<number>();
  for (const r of SEALING_RECORDS) {
    if (r.width === width && r.gusset === gusset) {
      set.add(r.sealPair[0]);
      set.add(r.sealPair[1]);
    }
  }
  return [...set].sort((a, b) => b - a); // 降順
}
```
