## 3. コード分析で特定した問題

### 3.1 Critical: 商品タイプ定義の不整合

**問題 #1**: `compatibleWith`配列に存在しない商品タイプが含まれている

| compatibleWith | 実在する商品タイプ | 影響を受ける後加工 |
|----------------|-------------------|-------------------|
| `lap_seal` | ❌ 存在しない | glossy, matte, notch各種, valve |
| `soft_pouch` | ❌ 存在しない | corner-round, bottom-open |
| `heavy_duty` | ❌ 存在しない | hang-hole-8mm |

**推奨修正**:
```typescript
// processingConfig.ts内のcompatibleWithを修正
// 存在しない商品タイプを削除
```

---

**問題 #2**: 商品タイプ定義のファイル間不一致

| ファイル | 定義されている商品タイプ |
|--------|----------------------|
| `types/quote.ts` | flat_3_side, stand_up, gusset, lap_seal, box |
| `types/database.ts` | flat_3_side, stand_up, gusset, box, flat_with_zip, special, soft_pouch, spout_pouch, roll_film |
| `templates.ts` | flat_3_side, stand_up, box, box_with_valve, spout_pouch, flat_with_zip, roll_film |
| `product-data.ts` | flat_3_side, gassho, stand_up, box, spout_pouch, roll_film |

### 3.2 High: gussetとboxの混在

**問題 #3**: `gusset`と`box`が同じ商品タイプ（ガゼットパウチ）を指しているが、コード内で混在して使用されている。

### 3.3 Critical: 厚さIDの命名規則の不統一

| 材質タイプ | 厚さID形式 | 例 |
|-----------|-----------|---|
| PET系, AL系 | light, medium, heavy, ultra | `light`, `medium`, `heavy`, `ultra` |
| NY/LLDPE系 | light_50, standard_70, heavy_90... | `light_50`, `standard_70`, `heavy_90`, `ultra_100`, `maximum_110` |
| Kraft系 | light_50, standard_70, heavy_90... | `light_50`, `standard_70`, `heavy_90`, `ultra_100`, `maximum_110` |

### 3.4 後加工オプションの互換性問題

**問題 #8**: `compatibleWith`配列内に、存在しない商品タイプへの参照が多数ある

## 5. 推奨修正プラン

### 5.1 Critical（優先度高）- 今週中に対応

1. **商品タイプ定義の統一**
   - 新規ファイル `types/product-types.ts` を作成
   - 全ての商品タイプIDを一箇所で定義
   - 他のファイルからはimportして使用

2. **compatibleWith配列の修正**
   - 存在しない商品タイプ(`lap_seal`, `soft_pouch`, `heavy_duty`)を削除
   - 正しい商品タイプIDに置き換え

3. **厚さID命名規則の統一**
   - 全ての材質で `light`, `medium`, `heavy`, `ultra` の4種類に統一

### 5.2 High（優先度高）- 今月対応

4. **gussetとboxの統一**
   - どちらか一方に統一して使用

5. **後加工オプションの排他制御**
   - UIコンポーネントでカテゴリー内の排他制御を実装

### 5.3 Medium（優先度中）- 次回リリース

6. **URLパラメータの明確化**
   - `/member/orders` にデフォルトで `?tab=active` を適用
   - または、URLパラメータなしでも全注文を表示するように改善

7. **ドキュメントの更新**
   - 商品タイプ、材質、後加工の仕様書を更新

### 5.4 Critical（緊急）- 日本語翻訳不整合の修正 ⚠️

8. **後加工オプション日本語翻訳の統一**
   - **問題**: 同じ後加工オプションがページによって異なる日本語で表示されている
   - **影響ファイル**: 13ファイル

     | top-open | 上端開封 | 上端開封, **上部開放**, **上部解放** |

     | valve-yes | バルブ付き | バルブ付き, **バルブあり**, バルブ |
     | hang-hole-6mm | 吊り下げ穴 (6mm) | 吊り下げ穴 (6mm), **吊り穴(6mm)** |

## 6. 結論

今回の検証で以下の結論を得ました：

1. **ページ間連動は正常に動作している**
   - 全ての管理者ページ、メンバーページが正常に機能している
   - データの受け渡し、画面遷移、ステータス更新が正しく動作している

2. **コードの一貫性に改善の余地がある**
   - 商品タイプ定義が複数のファイルに分散している
   - 厚さIDの命名規則が材質によって異なる
   - これらの統一により、保守性が向上する

3. **日本語翻訳の不整合がユーザー体験に影響** ⚠️
   - 後加工オプションの日本語表記がページによって異なる
   - 例: 「ジッパー付き」vs「チャック付き」、「上端開封」vs「上部開放」
   - 詳細レポート: `.omc/reports/translation-inconsistency-report.md`
   - これはユーザーに直接見える表示のため、優先的に修正することを推奨

4. **実際の問題は限定的**
   - compatibleWith配列の問題は、実際のページ動作には影響していない
   - これは、実際の商品タイプ選択肢に存在しないタイプが含まれているため
   - ただし、将来的な機能拡張時のバグを防ぐため、修正を推奨

**総合評価**: ワークフローは正常に動作しているが、**日本語翻訳の不整合がユーザー体験に直接影響している**。これを優先的に修正し、その後コード品質向上のための段階的な修正を推奨する。
