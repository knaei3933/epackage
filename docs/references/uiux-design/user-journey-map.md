# B2B 워크플로우 사용자 여정 맵 (User Journey Map)

## 1. 사용자 페르소나

### 1.1 주요 페르소나 정의

**페르소나 A: 구매 담당자 (신규 고객)**
- **이름:** 박지민 (34세, 여성)
- **직책:** 마케팅 팀 / 대리
- **회사:** 중소기업 (직원 50명)
- **목표:** 기념품 패키지 제작 의뢰
- **특징:**
  - B2B 주문 경험 부족
  - 시각적 커뮤니케이션 선호
  - 모바일 업무 빈도 높음
  - 즉각적인 피드백 기대

**페르소나 B: 구매 담당자 (기존 고객)**
- **이름:** 김철수 (45세, 남성)
- **직책:** 구매팀 / 과장
- **회사:** 대기업 (직원 500명)
- **목표:** 정기 패키지 발주
- **특징:**
  - B2B 주문 프로세스 익숙
  - 효율성과 신속성 중시
  - 대량 주문 경험
  - 정확한 납기 준수 요구

**페르소나 C: 관리자**
- **이름:** 이관리 (38세, 남성)
- **직책:** 생산관리팀 / 대리
- **목표:** 주문 효율적 처리
- **특징:**
  - 다중 주문 동시 처리
  - 상태 추적 중요
  - 자동화 선호


### 상세 여정 단계

#### Phase 1: 발견 & 회원가입

**단계 1.1: 홈페이지 방문**

| 요소 | 설명 |
|------|------|
| **진입점** | 검색 엔진, SNS 광고, 지인 추천 |
| **1차 목표** | 서비스 이해 및 신뢰 형성 |
| **주요 행동** |
| - 랜딩 페이지 스크롤 | - 서비스 소개 확인 |
| - 제품 카탈로그 탐색 | - 가격대 파악 |
| - 고객 사례 확인 | - 포트폴리오 검토 |
| **UX 목표** | 3분 내에 서비스 가치 이해 |
| **감정 상태** | 호기심 → 관심 → 신뢰 형성 중 |
| **페인 포인트** |
| - 너무 많은 정보 | - 어디서 시작해야 할지? |
| - 기업 고객 여부 불확실 | - 내가 타겟인가? |
| **해결 방안** |
| - "기업 고객 전용" CTA 명확히 | - 간단한 퀴즈로 가이드 |
| - 핵심 가치 상단에 배치 | - 고객 사례 강조 |

**단계 1.2: 회원가입 결정**

| 요소 | 설명 |
|------|------|
| **트리거** | "회원가입" 또는 "견적 요청" CTA 클릭 |
| **1차 목표** | 회원가입 완료 |
| **주요 행동** |
| - 회원가입 페이지 진입 | - 기업 정보 입력 |
| - 사업자등록번호 확인 | - 담당자 정보 입력 |
| - 이용약관 동의 | - 가입 완료 |
| **UX 목표** | 5분 내에 가입 완료 |
| **감정 상태** | 기대 → 약간의 번거로움 → 안도 |
| **페인 포인트** |
| - 사업자등록번호 찾기 | - 입력 항목 많음 |
| - 약관 길고 복잡 | - 필수/선택 구분 불명확 |
| **해결 방안** |
| - 사업자등록번호 자동 검증 API | - 필수 항목만 최소화 |
| - 약관 요약본 제공 | - 진척도 표시 (Step 1/3) |

**단계 1.3: 첫 로그인 & 대시보드**

| 요소 | 설명 |
|------|------|
| **트리거** | 가입 완료 후 자동 로그인 |
| **1차 목표** | 포털 이해 및 다음 단계 파악 |
| **주요 행동** |
| - 대시보드 탐색 | - 메뉴 구조 파악 |
| - "새 견적 요청" CTA 발견 | - 가이드 투어 수령 (선택) |
| **UX 목표** |
| - 첫 번째 핵심 액션 유도 (견적 요청) |
| - 허겁지겁 방지 (명확한 다음 단계) |
| **감정 상태** | 성취감 → 흥미 → 준비 완료 |
| **페인 포인트** |
| - 빈 페이지 혼란 | - 무엇부터 해야 할지? |
| - 너무 많은 메뉴 | - 정보 과부하 |
| **해결 방안** |
| - Empty State with CTA | - 온보딩 체크리스트 |
| - 단계별 가이드 제공 | - 인터랙티브 투어 |

#### Phase 2: 견적 요청

**단계 2.1: 견적 요청 폼 진입**

| 요소 | 설명 |
|------|------|
| **진입 경로** |
| - 대시보드 "새 견적 요청" 버튼 |
| - 사이드바 "견적 관리" → "견적 요청" |
| **1차 목표** |
| - 원하는 제품 찾기 |
| - 기본 사양 입력 |
| **주요 행동** |
| - 제품 카탈로그 탐색 | - 필터 적용 (크기, 재질, 가격) |
| - 제품 선택 | - 수량 입력 |
| - 사양 입력 | - 추가 요구사항 메모 |
| **UX 목표** |
| - 10분 내에 견적 요청 완료 |
| - 재설정 최소화 (저장 기능) |
| **감정 상태** |
| - 흥미 → 집중 → 약간의 피로감 → 기대 |
| **페인 포인트** |
| - 용어 이해 어려움 | - "인쇄 방식", "코팅 종류" 등 |
| - 너무 많은 옵션 | - 선택 장애 |
| - 파일 첨부 어려움 | - 크기 제한, 형식 제한 |
| **해결 방안** |
| - 용어 설명 툴팁 | - 도움말 모달 |
| - 추천 옵션 프리셋 | - "인기있는 조합" |
| - 드래그 앤 드롭 업로드 | - 진척도 표시 |
| - 임시 저장 기능 | - "나중에 계속하기" |

**단계 2.2: 견적 요청 제출**

| 요소 | 설명 |
|------|------|
| **트리거** |
| - "견적 요청 제출" 버튼 클릭 |
| - 검토 팝업 표시 |
| **주요 행동** |
| - 입력 내용 검토 | - 수정 필요 시 뒤로 가기 |
| - 제출 확인 | - 완료 메시지 |
| **UX 목표** |
| - 제출 전 오류 최소화 |
| - 다음 단계 기대감 형성 |
| **감정 상태** |
| - 긴장 → 안도 → 기대 |
| **페인 포인트** |
| - 제출 실패 공포 | - "다시 입력해야 하나?" |
| - 언제 답변 오나? | - 대기 시간 불확실 |
| **해결 방안** |
| - 검토 단계 강조 | - 마지막 수정 기회 |
| - 명확한 타임라인 | - "영업일 기준 1일 이내" |
| - 자동 이메일 발송 | - 즉시 확인 |

**단계 2.3: 대기 기간**

| 요소 | 설명 |
|------|------|
| **지속 시간** | 1-2 영업일 |
| **고객 행동** |
| - 이메일 확인 | - 알림 설정 |
| - 포털 주기적 방문 | - 상태 확인 |
| - 경쟁사 견적 비교 (경우 있음) |
| **UX 목표** |
| - 투명한 프로세스 |
| - 불안감 최소화 |
| **감정 상태** |
| - 기대 → 불안 → 초조 |
| **페인 포인트** |
| - 침묵 기간 | - 진행 상황 모름 |
| - 연락 없음 | - "내 요청이 도착했나?" |
| **해결 방안** |
| - 자동 확인 이메일 (즉시) | - 요청 접수 알림 |
| - 상태 업데이트 이메일 | - "견적 작성 중" |
| - 채팅봇 상담 | - 즉각적인 응대 |

#### Phase 3: 견적 검토 & 주문

**단계 3.1: 견적서 수신**

| 요소 | 설명 |
|------|------|
| **알림 방식** |
| - 이메일 (견적서 PDF 첨부) |
| - SMS/알림 (선택) |
| - 포털 내 알림 |
| **주요 행동** |
| - 이메일 확인 | - 첨부파일 다운로드 |
| - 포털 로그인 | - 견적 상세 페이지 진입 |
| **UX 목표** |
| - 견적서 쉽게 이해 |
| - 액션 유도 (승인/거절) |
| **감정 상태** |
| - 흥미 → 검토 → 결정 |
| **페인 포인트** |
| - PDF 복잡함 | - 어디서 봐? |
| - 가격 이해 어려움 | - 단가, 수량, 합계 |
| - 납기일 기한 | - "우리에게 너무 빨라?" |
| **해결 방안** |
| - 견적 상세 페이지 (Web) | - PDF보다 더 나은 UX |
| - 가격 breakdown | - 항목별 상세 |
| - 납기일 선택지 | - "긴급", "표준", "여유" |
| - 1:1 채팅 | - 질문 즉시 해결 |

**단계 3.2: 견적 승인**

| 요소 | 설명 |
|------|------|
| **트리거** |
| - "견적 승인하기" 버튼 클릭 |
| - 승인 확인 팝업 |
| **주요 행동** |
| - 최종 검토 | - 주문 전환 확인 |
| - 결제 방법 선택 (필요 시) | - 주문 확정 |
| **UX 목표** |
| - 쉽고 빠른 주문 전환 |
| - 후회 최소화 |
| **감정 상태** |
| - 결심 → 확신 → 기대 |
| **페인 포인트** |
| - 되돌릴 수 없음 | - "승인 후 취소 가능?" |
| - 갑작스러운 질문 | - 추가 정보 요청 |
| **해결 방안** |
| - 취소 정책 명시 | - "승인 후 24시간 이내 취소 가능" |
| - FAQ 섹션 | - 자주 묻는 질문 |
| - 상담원 연결 | - 실시간 채팅 |

**단계 3.3: 주문 확정**

| 요소 | 설명 |
|------|------|
| **완료 피드백** |
| - 주문번호 부여 | - "주문 #ORD-2024-001 생성 완료" |
| - 다음 단계 안내 | - "데이터 업로드 페이지로 이동" |
| - 이메일 확인 | - 주문서 PDF 발송 |
| **UX 목표** |
| - 성취감 강화 |
| - 다음 단계 명확히 |
| **감정 상태** |
| - 성취감 → 안도 → 기대 |
| **페인 포인트** |
| - "이제 뭐 하지?" | - 다음 단계 불명확 |
| - 너무 많은 정보 | - 메일, SMS, 포털 알림 |
| **해결 방안** |
| - 단일 CTA | - "데이터 업로드하기" |
| - 체크리스트 제공 | - "다음 5단계" |
| - 진척도 표시 | - "1/10 단계 완료" |

#### Phase 4: 데이터 제공

**단계 4.1: 디자인 파일 준비**

| 요소 | 설명 |
|------|------|
| **사전 요구사항** |
| - 디자이너에게 파일 요청 |
| - .ai 또는 PDF 파일로 저장 |
| - 파일 사이즈 확인 (50MB 이하) |
| **고객 행동** |
| - 내부 디자이너 협의 | - 파일 준비 |
| - 요구사항 확인 | - 포털 방문 |
| **UX 목표** |
| - 파일 요구사항 명확히 |
| - 업로드 간단하게 |
| **감정 상태** |
| - 준비 → 긴장 → 업로드 |
| **페인 포인트** |
| - 파일 형식 모름 | - ".ai가 뭐지?" |
| - 파일 너무 큼 | - "어떻게 줄이지?" |
| - 여러 파일 | - "어떤 파일 올리지?" |
| **해결 방안** |
| - 파일 가이드 제공 | - "준비 체크리스트" |
| - 파일 압축 가이드 | - "ZIP으로 압축하기" |
| - 미리보기 기능 | - 업로드 전 확인 |

**단계 4.2: 포털 업로드**

| 요소 | 설명 |
|------|------|
| **진입** |
| - 주문 상세 페이지 |
| - "데이터 업로드" 버튼 |
| - 데이터 업로드 페이지 |
| **주요 행동** |
| - 파일 선택 | - 드래그 앤 드롭 |
| - 업로드 진행 | - 프로그레스 바 |
| - 완료 확인 | - 성공 메시지 |
| **UX 목표** |
| - 3분 내에 업로드 완료 |
| - 실시간 피드백 |
| **감정 상태** |
| - 긴장 → 안도 → 흥미 |
| **페인 포인트** |
| - 업로드 실패 | - "다시 해야 하나?" |
| - 느린 속도 | - "시간이 너무 오래 걸려" |
| - 에러 메시지 복잡 | - "무슨 말이야?" |
| **해결 방안** |

## 5. 페인 포인트 & 해결 방안

### 5.1 주요 페인 포인트 상세

- 신규 고객에게 너무 많은 정보
- 어디서부터 시작해야 할지 모름
- 기업 고객 여부 불확실

**해결 방안:**
- 랜딩 페이지 단순화
- "기업 고객 전용" 명확한 CTA
- 인터랙티브 퀴즈로 가이드
- 온보딩 투어 제공

#### PP2: 용어 이해 어려움 (Phase 2, 5)

**문제:**
- B2B 특화 용어 이해 부족
- "인쇄 방식", "코팅 종류" 등
- 작업표준서, 계약서 용어

**해결 방안:**
- 툴팁 설명 제공
- 도움말 모달
- 용어 사전
- 평이한 언어 번역

#### PP3: 대기 기간 불안 (Phase 5, 11)

**문제:**
- 견적, 생산 대기 기간의 불안감
- 진행 상황 모름
- 침묵 기간

**해결 방안:**
- 자동 확인 이메일 (즉시)
- 상태 업데이트 이메일
- 실시간 진척도 표시
- 주기적 업데이트

#### PP4: 복잡한 서명 (Phase 10)

**문제:**
- 전자 서명 프로세스 복잡
- 법적 효력 우려
- 서명 자신 없음

**해결 방안:**
- 간단한 서명 UI
- 여러 번 시도 가능
- 법적 효력 안내
- 타임스탬프 기록

1. **온보딩 개선**
   - 인터랙티브 투어 추가
   - 첫 번째 핵심 액션 가이드
   - Empty State CTA 강화

2. **알림 시스템 강화**
   - 자동 확인 이메일
   - 상태 업데이트 알림
   - SMS 알림 (선택)

3. **진척도 투명성**
   - 실시간 업데이트
   - 예상 완료일 표시
   - 마지막 업데이트 시간

### 6.2 중기 개선 (3-6개월)

1. **AI 추출 정확도**
   - 머신러닝 모델 개선
   - 사용자 피드백 수집
   - 정확도 % 표시

2. **모바일 최적화**
   - 반응형 디자인
   - 모바일 앱 고려
   - 푸시 알림

3. **1:1 채팅**
   - 실시간 상담
   - 챗봇 자동 응대
   - 관리자 전달

### 6.3 장기 개선 (6-12개월)

1. **자동화**
   - 견적 자동 생성
   - 작업표준서 자동 생성
   - 계약서 자동 생성

2. **예측**
   - 납기일 예측
   - 가격 예측
   - 생산 일정 최적화

3. **개인화**
   - 추천 제품
   - 맞춤형 사양
   - 프로젝트 템플릿
