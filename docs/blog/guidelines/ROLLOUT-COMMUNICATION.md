## 🎯 主な特徴

### 1. 包括的なコンテンツ戦略
- **17記事タイプ完全対応**: 製品紹介、実践的ノウハウ、印刷技術、導入事例
- **5種類のテンプレート**: 各記事タイプに最適化された執筆フォーマット
- **詳細な構成ガイド**: リード文からまとめまでのベストプラクティス

### 2. Google & Naver SEO対応
- **統合SEO戦略**: 両プラットフォームで最適化されたチェックリスト
- **競合解決フレームワーク**: プラットフォーム間の要件競合を解決
- **最新アルゴリズム対応**: E-E-A-T、Core Web Vitals対応

### 3. nanobanana MCP 画像生成
- **50+プロンプトテンプレート**: 記事別の最適化されたプロンプト
- **画像サイズ規格**: Hero、Section、Product、Comparison等
- **品質ガイドライン**: ファイル形式、圧縮、Altテキスト

### 4. 品質管理プロセス
- **公開前チェックリスト**: SEO、技術、品質の包括的確認
- **専門家レビュー**: 専門性担保のためのプロセス
- **変更管理**: 定期レビューと緊急対応

## 🚀 はじめに

### ステップ1: メインガイドを読む
**EXPERT_BLOG_WRITING_GUIDE.md** を最初から最後までお読みください。約30分で全体像を把握できます。

### ステップ2: 記事テンプレートを確認
担当する記事タイプのテンプレートを確認し、構成を理解してください。

### ステップ3: SEOチェックリストを確認
記事公開前に `integrated-seo-checklist.md` を使用してSEO対応を確認してください。

### ステップ4: 画像生成に取り組む
`blog-image-nanobanana-guide.md` を参照して、nanobanana MCPを使用した画像生成を行ってください。

## 📚 クイックリファレンス

### タイトル作成
```
【カテゴリー】メインキーワード｜サブタイトル（魅力を伝える）

例: 【製品紹介】スタンドパウチ｜陳列効果を最大化する自立袋
```

### メタディスクリプション
```
120-160文字で、読者のメリットと記事内容を簡潔に説明。
```

### 記事構成（製品紹介）
```
1. リード文（読者の悩み）
2. 製品定義（基本情報）
3. メリット・デメリット
4. 主な用途
5. 選び方のポイント
6. 発注から納品まで
7. まとめ
```

### 文字数目安
- 製品紹介: 3,000-5,000文字
- 実践的ノウハウ: 4,000-7,000文字
- 導入事例: 5,000-10,000文字
