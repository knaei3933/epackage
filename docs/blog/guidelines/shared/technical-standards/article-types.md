# Article Types and Categories
# 記事タイプとカテゴリー設計

> **対象**: ライター、編集者
> **目的**: 記事の種類と分類を理解する
> **適用**: 全言語共通

---

## 記事タイプ別ガイドライン

### A. 製品紹介記事 (product-intro)

**目的**: 製品の特徴・メリットを伝え、問い合わせを獲得

**基本構造**:
```markdown
1. リード文（読者の悩み）
2. 製品定義（基本情報）
3. メリット・デメリット
4. 活用シーン・用途
5. 選び方のポイント
   - サイズ選定
   - 素材選定
   - オプション選定
6. 発注フロー（STEP形式）
7. まとめ
8. CTA
```

**トーン**: 専門的かつ親しみやすく、実用性重視

**文字数**: 3,000〜5,000文字

---

### B. 実践的ノウハウ記事 (practical-tips)

**目的**: 読者の課題解決に役立つ実践的知識を提供

**基本構造**:
```markdown
1. リード文（読者の状況・悩み）
2. ノウハウの定義・基本
3. 具体的手法・テクニック
4. 実践ステップ
5. 成功事例・失敗事例
6. チェックリスト
7. まとめ
8. CTA
```

**トーン**: 教育的、具体的、アクション可能

**文字数**: 4,000〜7,000文字

---

### C. 印刷技術記事 (printing-tech)

**目的**: 専門知識を分かりやすく解説し、信頼性を構築

**基本構造**:
```markdown
1. リード文（技術への関心）
2. 技術の定義・歴史
3. 仕組み・原理
4. メリット・デメリット
5. 比較検討
6. 活用事例
7. まとめ
8. CTA
```

**トーン**: 専門的、分かりやすい、客観的

**文字数**: 2,500〜4,000文字

---

### D. 導入事例記事 (customer-stories)

**目的**: お客様の成功事例を通じて、信頼性を構築

**基本構造**:
```markdown
1. リード文（読者の課題）
2. 会社概要・導入背景
3. 導入前の課題
4. 選定理由・決定要因
5. 具体的活用方法
6. 成果と効果
7. 今後の展望
8. まとめ
9. CTA
```

**トーン**: 語り口調、リアリティのあるお客様の声

**文字数**: 5,000〜10,000文字

## 記事タイプ選定ガイド

### 製品紹介記事を使用する場合

- 新製品の紹介
- 製品の詳細な説明が必要
- 読者が製品の購入を検討している

### 実践的ノウハウ記事を使用する場合

- 読者の具体的な課題を解決したい
- 実践的な手法を紹介したい
- アクション可能なアドバイスを提供したい

### 印刷技術記事を使用する場合

- 専門知識を分かりやすく解説したい
- 技術的な信頼性を高めたい
- 比較検討資料を提供したい

### 導入事例記事を使用する場合

- お客様の成功事例を紹介したい
- 具体的な導入効果を示したい
- 信頼性を構築したい
