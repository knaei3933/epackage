# 専門家レビュースケジュール

> **目的**: ブログコンテンツの専門性と正確性を確保するための専門家レビュープロセス
> **更新日**: 2025-04-11
> **バージョン**: 1.0

---

## 概要

本スケジュールは、Epackage Labブログコンテンツの専門家レビューを効率的に実施するための枠組みを定義します。

---

## レビュアーの役割と責任

### 専門家レビュアー（技術）

**担当領域**:
- 製品仕様の正確性
- 技術用語の適切性
- 製造プロセスの記述
- 素材・印刷技術の正確性

**責任**:
- 技術的な正確性を保証
- 専門用語の適切な使用を確認
- 誤解を招く表現を修正

### 専門家レビュアー（法務）

**担当領域**:
- 法規制対応の確認
- 知的財産権の侵害リスク
- 競合他社への言及の適切性

**責任**:
- 法的リスクを評価
- 必要な免責事項を確認
- コンプライアンスを保証

### SEOレビュアー

**担当領域**:
- キーワードの適切な配置
- メタデータの最適化
- 構造化データの実装
- 内部リンクの適切性

**責任**:
- SEO効果を最大化
- 検索エンジンのガイドラインに準拠
- パフォーマンスを追跡

---

## レビューフェーズ

### フェーズ1: 下書きレビュー（必須）

**タイミング**: コンテンツ作成後、公開前

**実施内容**:
- [ ] 技術的正確性の確認
- [ ] 法的リスクの評価
- [ ] SEO最適化の確認
- [ ] 品質チェックリストの実施

**所要時間**: 2-3営業日

**責任者**: コンテンツ作成者

### フェーズ2: 公開前レビュー（推奨）

**タイミング**: 公開直前（公開1週間前）

**実施内容**:
- [ ] 最終品質確認
- [ ] 画像の表示確認
- [ ] リンク切れ確認
- [ ] モバイル表示確認

**所要時間**: 1営業日

**責任者**: コンテンツマネージャー

### フェーズ3: 定期レビュー（四半期）

**タイミング**: 四半期ごと

**実施内容**:
- [ ] 情報の鮮度確認
- [ ] 会社情報との整合性確認
- [ ] パフォーマンス評価
- [ ] 改善提案の作成

**所要時間**: 1週間

**責任者**: コンテンツ戦略担当者

## レビューチェックリスト

### 専門家レビュアー（技術）

| カテゴリー | チェック項目 | 確認内容 |
|-----------|-------------|----------|
| **製品情報** | 仕様 | 最小ロット、納期、価格が正しいか |
| | 素材 | 素材構成の記述が正確か |
| | 機能 | 製品機能の説明が正確か |
| **技術用語** | 用語 | 専門用語が適切に使われているか |
| | 定義 | 専門用語に定義が必要か |
| **製造プロセス** | 工程 | 製造工程の記述が正確か |
| | 納期 | 納期の記述が現実的か |

### 専門家レビュアー（法務）

| カテゴリー | チェック項目 | 確認内容 |
|-----------|-------------|----------|
| **知的財産** | 商標 | 競合他社の商標を侵害していないか |
| | 著作権 | 画像・テキストの権利を確認 |
| **法規制** | 食品表示法 | 必要な表示が含まれているか |
| | PL法 | 免責事項が適切か |
| **表現** | 誇大表現 | 過度な表現がないか |
| | 比較表現 | 競合他社との比較が適切か |

### SEOレビュアー

| カテゴリー | チェック項目 | 確認内容 |
|-----------|-------------|----------|
| **キーワード** | タイトル | メインキーワードが含まれているか |
| | メタディスクリプション | キーワードが適切に配置されているか |
| | 本文 | 自然な分散を維持しているか |
| **構造化データ** | JSON-LD | 正しく実装されているか |
| | パンくず | 正しく実装されているか |
| **内部リンク** | 関連記事 | 3本以上の関連記事へのリンクがあるか |
| | アンカーテキスト | 適切なアンカーテキストか |
| **パフォーマンス** | ページ速度 | Core Web Vitalsが良好か |
| | モバイル | モバイル対応が完備されているか |

## フィードバック統合プロセス

### フィードバックの収集

1. **フィードバックフォーム**
   ```markdown
   # レビューフィードバック
   
   ## 記事情報
   - タイトル: [記事タイトル]
   - URL: [記事URL]
   - カテゴリー: [カテゴリー]
   
   ## フィードバック
   
   ### 技術的な正確性
   - [ ] 問題なし
   - [ ] 修正必要
   - コメント: [具体的な指摘事項]
   
   ### 法的リスク
   - [ ] 問題なし
   - [ ] 修正必要
   - コメント: [具体的な指摘事項]
   
   ### SEO最適化
   - [ ] 問題なし
   - [ ] 改善余地あり
   - コメント: [具体的な改善提案]
   ```

2. **レビューシステム**
   - Google Docsのコメント機能
   - GitHubのPull Requestレビュー
   - プロジェクト管理ツールのタスク機能

### フィードバックの統合

1. **優先順位付け**
   - 高：技術的正確性、法的リスク
   - 中：SEO最適化、品質
   - 低：文体の調整、微細な修正

2. **対応**
   - 修正を実施
   - 修正内容をレビュアーに確認
   - 承認後に公開

## イベント駆動エンゲージメントモデル

### 定期イベント

| イベント | 頻度 | 参加者 | 目的 |
|--------|------|--------|------|
| **四半期レビュー** | 四半期ごと | 全レビュアー | コンテンツ全体の品質評価 |
| **月定例ミーティング** | 月次 | コンテンツチーム | 進捗確認と課題共有 |
| **週次進捗確認** | 週次 | コンテンツ作成者 | 作業進捗の確認 |

### トリガーイベント

| トリガー | アクション | 参加者 |
|---------|----------|--------|
| **重大な誤情報発見** | 即時レビュー | 専門家レビュアー |
| **法規制の変更** | 影響評価 | 法務レビュアー |
| **SEOアルゴリズム更新** | 影響評価と対応 | SEOレビュアー |
| **問い合わせの増加** | コンテンツ評価 | 全レビュアー |

## レビュー依頼のタイミング

### 標準的なスケジュール

| コンテンツタイプ | 下書き完了 | レビュー依頼 | レビュー完了 | 公開予定 |
|---------------|-----------|-----------|-----------|---------|
| **製品紹介** | 月曜 | 火曜 | 金曜 | 翌週月曜 |
| **実践的ノウハウ** | 月曜 | 火曜 | 金曜 | 翌週月曜 |
| **導入事例** | 月曜 | 火曜 | 翌週水曜 | 翌週金曜 |
| **業界ニュース** | 随時 | 即時 | 24時間以内 | 48時間以内 |

### 緊急レビュー

**条件**:
- 時事性の高いトピック
- 重要な法規制の変更
- 重大な誤情報の修正

**プロセス**:
1. 緊急レビューを依頼（理由を明記）
2. 優先的に対応（24時間以内）
3. 結果をステークホルダーに共有
