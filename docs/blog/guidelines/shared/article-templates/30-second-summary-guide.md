# 30秒でこの記事を理解する セクション作成ガイド
# 30초 요약 섹션 작성 가이드

> **対象**: ライター、編集者
> **目的**: 記事の核要要を簡潔に伝え、読者の理解度を向上させる
> **適用**: すべての記事タイプ
> **言語**: 日本語・韓国語共通

---

## 概要

**30秒でこの記事を理解する**セクションは、記事の最後に配置する要約セクションです。読者が記事全体を読む時間がない場合でも、核要なポイントを素早く理解できます。

---

## Brixaの事例

### 効果的な30秒要約の例

```
30秒でこの記事を理解する

· スタンドパウチは底面がW字型に折りたたまれ、内容物を入れると自立する袋です
· 段陳効果が高く、スペースを節約できます
· 内容物に応じて適切なPE層の厚さが異なります
· 100〜500gはPE40、500〜800gはPE60を推奨します
· 素材選択は「保存期間」と「内容物特性」を基準にします
```

**特徴**:
- 5-7個の要点
- 箇条書き形式
- 具体的な数値とアクション可能な情報
- 読者がすぐに実践できる内容

---

## 作成順序

### ステップ1: 記事完成後に作成

1. 記事全体を完成させる
2. 各セクションから重要なポイントを抽出
3. 5-7個の要点にまとめる

### ステップ2: ポイントの選定基準

**含めるべき内容**:
- 記事で学んだ主要な概念
- 実践可能なアクションアイテム
- よくある誤解や間違い
- 次のステップへの接続

**含めない内容**:
- 詳細な説明
- 補足情報
- 専門的な裏付け

### ステップ3: 箇条書き形式

```markdown
· ポイント1
· ポイント2
· ポイント3
```

**ルール**:
- 中点（・）で開始
- 各ポイントは1文〜2文
- 具体的な数値を含める
- 専門用語は平易に言い換え

## 記事タイプ別ガイド

### 製品紹介記事

**含めるポイント**:
1. 製品の定義
2. 主要な特徴
3. 選択基準
4. 具体的な数値
5. 次のアクション

**例**:
```
30秒でこの記事を理解する

· スタンドパウチは自立する袋型パッケージです
· 底面がW字型に折りたたまれ、内容物を入れると自立します
· 段陳効果が高く、商品の魅力を最大化できます
· 内容物100〜500gにはPE40、500〜800gにはPE60が適しています
· 最低500枚から製造可能で、納期は最短21日です
```

### 実用ノウハウ記事

**含めるポイント**:
1. 解決する問題
2. 具体的な解決策
3. 実践ステップ
4. 注意点
5. 期待できる効果

**例**:
```
30秒でこの記事を理解する

· パッケージデザインで商品の売上を2倍にする方法があります
· 最初にターゲット顧客を明確に定義します
· 競合他社と差別化できる色彩戦略を立てます
· デザイン案を作成し、実際の棚でテストします
· 専門家によるレビューで失敗リスクを最小化できます
```

### 技術解説記事

**含めるポイント**:
1. 技術の定義
2. 主要な種類
3. 選択基準
4. 比較結果
5. 推奨事項

**例**:
```
30秒でこの記事を理解する

· バリア性は酸素や湿気を遮断する機能です
· 代表的なバリア材はAL、VMPET、LLDPEの3種類です
· 保存期間3ヶ月未満ならLLDPE、3ヶ月以上ならALまたはVMPETを選びます
· ALは最強ですがコストが高く、VMPETはバランスが取れています
· 内容物と保存期間を基準に選択することが重要です
```

### 導入事例記事

**含めるポイント**:
1. 企業の課題
2. 導入ソリューション
3. 実施プロセス
4. 定量的成果
5. 学んだ教訓

**例**:
```
30秒でこの記事を理解する

· A社は陳列棚での商品訴求力不足に課題を持っていました
· スタンドパウチへの切り替えで視認性を向上させました
· デザイン刷新から3ヶ月で売上が1.5倍に増加しました
· 顧客アンケートで「商品が目立つ」評価が80%に上昇
· パッケージ変更は小さな投資で大きな効果を生み出します
```

### ビジネス視点記事

**含めるポイント**:
1. 問題の定義
2. 影響の数値
3. 根本的な原因
4. 解決策
5. アクションプラン

**例**:
```
30秒でこの記事を理解する

· 在庫は食品メーカーの利益を最大30%まで圧迫します
· 1SKUあたり月10万円の保管コストが発生します
· 原因は「大ロット生産モデル」と「多SKU化」の矛盾です
· 検証型開発で小ロットから始め、ヒットしたら増産します
· 在庫リスクを最小化しながら売上を最大化できます
```

## 良い例と悪い例

### 悪い例1: 具体性が不足

```
この記事ではスタンドパウチについて説明しました。
スタンドパウチは良いです。
```

**問題点**:
- 具体的な情報がない
- 読者が得られる価値が不明確
- アクション可能ではない

### 悪い例2: 長すぎる

```
30秒でこの記事を理解する

· スタンドパウチは底面がW字型に折りたたまれた構造を持ち、内容物を充填すると自立性を発揮する柔軟包装容器の一種です
· （以下、長文が続く）
```

**問題点**:
- 1文が長すぎる
- 要約ではなく要約版になっていない
- 専門用語が多すぎる

### 良い例1: 具体的で簡潔

```
30秒でこの記事を理解する

· スタンドパウチは底がW字型に折りたたまれ、中に物を入れると立つ袋です
· 段陳に強く、商品の顔が常に見えます
· 100〜500gには薄手、500〜800gには厚手を選びます
· 最低500枚から作れて、最短21日で納品されます
· デザインの無料チェックも受けられます
```

**特徴**:
- 平易な言葉
- 具体的な数値
- 明確なアクション
- 5つの要点

## 配置場所と形式

### 配置場所

```markdown
# 【記事タイプ】製品名｜サブタイトル

[リード文]

[目次]

[各セクション]

## まとめ

[まとめ本文]

---

## 30秒でこの記事を理解する

· 要点1
· 要点2
· 要点3
```

**ルール**:
- 記事の最後に配置
- 「まとめ」セクションの後に配置
- 水平線（---）で区切る

### 形式

```markdown
## 30秒でこの記事を理解する

· 要点1
· 要点2
· 要点3
· 要点4
· 要点5
```

**言語別タイトル**:
- 日本語: `## 30秒でこの記事を理解する`
- 韓国語: `## 30초 만에 이 기사를 이해하기`

## ライティングのコツ

### コツ1: 能動態を使う

**悪い例**:
```
· 推奨されます
· 考慮する必要があります
```

**良い例**:
```
· 推奨します
· 考慮してください
```

### コツ2: 具体的な数値を含める

**悪い例**:
```
· 適切な量を選びます
· 期間が短縮されます
```

**良い例**:
```
· 100〜500gにはPE40を選びます
· 納期が最短21日に短縮されます
```

### コツ3: 読者の視点に立つ

**悪い例**:
```
· 自社のサービスの特徴です
· 私たちが提供する機能です
```

**良い例**:
```
· あなたの商品の魅力を最大化します
· 失敗リスクを最小化できます
```

### コツ4: 専門用語を平易に言い換える

**悪い例**:
```
· ヒートシール性が優れています
· ガスバリア性が高いです
```

**良い例**:
```
· 熱でしっかり封できます
· 酸素や湿気をしっかり防ぎます
```

## トラブルシューティング

### 問題1: 要点が多すぎる

**解決策**:
- 類似した要点を統合する
- 優先順位をつけてトップ5-7に絞る
- 補足的な要点は削除する

### 問題2: 要点が抽象的すぎる

**解決策**:
- 具体的な数値を追加する
- 「何を」「どうする」を明確にする
- 読者にとっての利益を明示する

### 問題3: 要点が長すぎる

**解決策**:
- 1文を2文に分ける
- 詳細は記事本文に譲る
- キーワードだけを抽出する
