# プラットフォーム別SEO意思決定フレームワーク

> **目的**: GoogleとNaverのSEO要件の競合を解決するための意思決定フレームワーク
> **更新日**: 2025-04-11
> **バージョン**: 1.0

---

## 概要

Epackage Labのブログコンテンツは、日本市場（Google）と韓国市場（Naver）の両方で検索流入を最大化する必要があります。本フレームワークは、両プラットフォームのSEO要件の競合が発生した場合の意思決定プロセスを定義します。

---

## プラットフォーム特性比較

| 要件 | Google | Naver | 優先順位 |
|------|--------|-------|----------|
| **タイトル長さ** | 30-60文字（推奨40文字） | 40-70文字（推奨55文字） | Google優先 |
| **メタディスクリプション** | 120-160文字 | 150-200文字 | Google優先 |
| **キーワード配置** | 自然な分散、過度な使用回避 | 前半への配置重視 | Naver優先 |
| **構造化データ** | 必須（JSON-LD） | 推奨（Microdata） | Google優先 |
| **内部リンク** | 関連性重視 | アンカーテキスト一致度重視 | Naver優先 |
| **コンテンツ長さ** | 品質重視（長すぎないこと） | 1,500文字以上推奨 | Naver優先 |
| **画像ALT** | 簡潔な説明 | キーワードを含めた詳細説明 | Naver優先 |
| **更新頻度** | 鮮度より品質 | 定期的な更新を評価 | Naver優先 |

---

## 競合解決プロトコル

### ステップ1: 競合の識別

コンテンツ作成時に以下のチェックリストを実施：

- [ ] タイトル長さの競合（Google < 40文字 vs Naver > 55文字）
- [ ] キーワード配置方針の競合
- [ ] コンテンツ長さの競合
- [ ] 内部リンク戦略の競合
- [ ] 画像ALTテキスト方針の競合

### ステップ2: 影響度評価

| 影響レベル | Google | Naver | 決定 |
|-----------|--------|-------|------|
| **高** | トラフィックの70%以上 | トラフィックの30%未満 | Google優先 |
| **中** | トラフィックの50-70% | トラフィックの30-50% | 妥協案検討 |
| **低** | トラフィックの50%未満 | トラフィックの50%以上 | Naver優先 |

### ステップ3: 妥協案の作成

影響レベルが「中」の場合、以下の妥協案を検討：

#### タイトル長さの妥協
```
基本方針: Googleの推奨（30-60文字）をベースに、Naverの要求（40-70文字）を満たす範囲で調整

最適解: 50文字前後（両方の推奨範囲に収まる）
```

#### キーワード配置の妥協
```
基本方針: 前半にキーワードを配置（Naver要件）、自然な分散を維持（Google要件）

実装: 
- 最初の段落にメインキーワードを含める
- H2見出しに関連キーワードを配置
- 全体的に自然な分散を維持
```

#### コンテンツ長さの妥協
```
基本方針: 品質を維持しつつ、Naverが好む1,500文字以上を目指す

実装:
- 製品紹介: 3,000-5,000文字
- 実践的ノウハウ: 4,000-7,000文字
- 導入事例: 5,000-10,000文字
```

## プラットフォーム別最適化ガイド

### Google最適化（日本市場）

#### 優先事項
1. E-E-A-T（経験・専門性・权威性・信頼性）
2. 構造化データ（JSON-LD）
3. モバイルファーストインデックス
4. Core Web Vitals
5. 自然言語処理（BERT）

#### 実装チェックリスト
- [ ] Schema.orgのJSON-LDを実装
- [ ] Core Web Vitalsを良好に維持
- [ ] モバイル対応を完備
- [ ] ページ読み込み速度 < 2.5秒
- [ ] HTTPS対応
- [ ] サイトマップを送信

### Naver最適化（韓国市場）

#### 優先事項
1. キーワード密度（2-3%）
2. コンテンツ長さ（1,500文字以上）
3. 定期的な更新
4. ソーシャルシグナル
5. Naver Cafe/Blog連携

#### 実装チェックリスト
- [ ] メインキーワードを最初の段落に配置
- [ ] 1,500文字以上のコンテンツ
- [ ] 週1回以上の更新
- [ ] Nver Analytics導入
- [ ] Nver Search Advisor登録
- [ ] Nver Cafe/Blogでの拡散

## 意思決定ツリー

```
START
  │
  ├─ 競合がある？
  │   ├─ いい → 影響度評価へ
  │   └─ いいえ → プラットフォーム別ガイド適用
  │
  ├─ 影響度評価
  │   ├─ Google優先（高） → Google要件を適用
  │   ├─ Naver優先（低） → Naver要件を適用
  │   └─ 妥協案検討（中） → 妥協案作成へ
  │
  ├─ 妥協案作成
  │   ├─ 妥協可能 → 統合ソリューション実装
  │   └─ 妥協不可 → A/Bテスト実施
  │
  └─ A/Bテスト
      ├─ 勝者決定 → 勝者の要件を適用
      └─ 有意差なし → Google要件を優先
```
