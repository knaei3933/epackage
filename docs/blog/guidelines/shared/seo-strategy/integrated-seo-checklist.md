# 統合SEOチェックリスト

> **目的**: GoogleとNaverの両方で最適化されたブログコンテンツを作成するための包括的チェックリスト
> **更新日**: 2025-04-11
> **バージョン**: 1.0

---

## 使用方法

本チェックリストは、記事公開前に必ず実施してください。各項目は「Google必須」「Naver必須」「両方推奨」のカテゴリーに分類されています。

---

## 1. 基本SEO要素

### タイトル
- [ ] **[両方推奨]** 長さが50文字前後（Google: 30-60文字、Naver: 40-70文字の overlap）
- [ ] **[Google必須]** メインキーワードをタイトルの先頭に配置
- [ ] **[Naver必須]** 記事カテゴリーをタイトルに含める（例: 【製品紹介】）
- [ ] **[両方推奨]** 魅力的なサブタイトルを含める（例: ｜陳列効果を最大化する自立袋）
- [ ] **[両方推奨]** 重複タイトルがないことを確認

### メタディスクリプション
- [ ] **[両方推奨]** 長さが120-160文字（Google推奨、Naverも許容）
- [ ] **[Google必須]** 読者のメリットを明確に記述
- [ ] **[Naver必須]** メインキーワードを2回以上含める
- [ ] **[両方推奨]** アクションを促す要素を含める（CTA誘導）

---

## 2. コンテンツ構造

### 見出し構造
- [ ] **[Google必須]** H1: 記事タイトル（1つのみ）
- [ ] **[Google必須]** H2: メインセクション（5-7個）
- [ ] **[両方推奨]** H3: サブセクション（適切に使用）
- [ ] **[Naver必須]** H2にメインキーワードを含める

### コンテンツ長さ
- [ ] **[Naver必須]** 総文字数1,500文字以上
- [ ] **[Google必須]** 品質を維持（無駄な長文化なし）
- [ ] **[両方推奨]** 適切な段落分割（2-4文で区切る）

### キーワード配置
- [ ] **[Naver必須]** 最初の段落にメインキーワードを含める
- [ ] **[Google必須]** キーワード密度1-2%（過度な使用回避）
- [ ] **[両方推奨]** 関連キーワードを自然に配置
- [ ] **[Naver必須]** H2見出しに関連キーワードを含める

---

## 3. 構造化データ

### JSON-LD
- [ ] **[Google必須]** BlogPostingスキーマを実装
- [ ] **[Google必須]** author、publisher、datePublishedを含める
- [ ] **[両方推奨]** mainEntityOfPageを含める
- [ ] **[両方推奨]** articleSection（カテゴリー）を指定
- [ ] **[両方推奨]** keywords（tags）を含める

### その他構造化データ
- [ ] **[Google必須]** BreadcrumbListを実装
- [ ] **[Naver推奨]** FAQPageがある場合は実装
- [ ] **[両方推奨]** Organizationスキーマをサイトに実装

---

## 4. 画像最適化

### ファイル名
- [ ] **[Naver必須]** ファイル名にキーワードを含める（例: stand-pouch-features.png）
- [ ] **[両方推奨]** 英数字とハイフンのみ使用

### ALTテキスト
- [ ] **[Google必須]** 簡潔な説明（10-15文字）
- [ ] **[Naver推奨]** キーワードを含めた説明（20-30文字）
- [ ] **[妥協案]** `alt="スタンドパウチの製品イメージ。底部がW字型に折りたたまれ、自立する様子を示す"`

### ファイルサイズ
- [ ] **[両方推奨]** Hero画像: 最大300KB
- [ ] **[両方推奨]** セクション画像: 最大200KB
- [ ] **[両方推奨]** サムネイル: 最大50KB

---

## 5. 内部リンク

### リンク戦略
- [ ] **[Google必須]** 関連記事3本へのリンク
- [ ] **[Naver必須]** アンカーテキストにキーワードを含める
- [ ] **[両方推奨]** 自然な文脈でのリンク配置

### パンくず
- [ ] **[Google必須]** パンくずリストを実装
- [ ] **[Naver推奨]** パンくずにキーワードを含める

---

- [ ] **[Google必須]** Core Web Vitals: CLS < 0.1

### モバイル

- [ ] **[両方推奨]** レスポンシブデザイン実装

## 8. コンテンツ品質

### E-E-A-T（Google）
- [ ] **[Google必須]** 専門家によるレビュー済み
- [ ] **[Google必須]** 引用・出典を明記
- [ ] **[Google必須]** 会社情報と連絡先を掲載

### 鮮度（Naver）
- [ ] **[Naver推奨]** 過去3ヶ月以内の更新
- [ ] **[Naver推奨]** 更新日を明記

### ゲーテスケジュール（SEO最適化）
- [ ] **[Google必須]** 同日に複数記事を公開していない
- [ ] **[Google必須]** 週1-2回の自然な間隔で公開
- [ ] **[Google必須]** 時間帯を多様化（午前/午後を混合）
- [ ] **[Google必須]** 過去の日付を現在の日付に修正
- [ ] **[Google必須]** 記事間の間隔を1-2週間に設定

**推奨ゲーテパターン**:
```
週1-2回、不規則なスケジュール:
- 月曜 10:15 / 水曜 14:20 / 金曜 09:30
- 時間帯: 09:00-11:30（午前）、14:00-17:00（午後）
- 間隔: 1-2週間
```

## 競合解決時の対応

### タイトル長さの競合
- [ ] 50文字前後に調整（両方の推奨範囲）
- [ ] メインキーワードは先頭に配置
- [ ] カテゴリーとサブタイトルを含める

### キーワード配置の競合
- [ ] 最初の段落にメインキーワードを配置（Naver優先）
- [ ] 全体的に自然な分散を維持（Google優先）
- [ ] H2見出しに関連キーワードを配置（両方対応）
