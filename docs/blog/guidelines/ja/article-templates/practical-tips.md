## 成功事例の書き方

**構造**:
- **課題**: 具体的な課題を説明
- **対策**: 講じた具体的な対策
- **結果**: 数値や評価を含む結果

### 例

```markdown
### 事例1: 小規模食品メーカーA社

**課題**:
- 既存パッケージが店頭で目立たない
- 競合商品との差別化ができていない

**対策**:
- スタンドパウチに変更（自立性アップ）
- ブランドカラーを統一
- 製品の特徴を伝えるキャッチコピー追加

**結果**:
- 店頭での目立ち度: 40%向上（店舗アンケート）
- 問い合わせ数: 月15件→25件に増加
- 販売数: 前年同期比+25%
```

## チェックリストの作成

記事の最後にチェックリストを追加すると、読者が実践しやすくなります。

### 例

```markdown
## チェックリスト

パッケージデザインを見直す際、以下の点をチェックしましょう。

- [ ] ターゲット読者を明確にしている
- [ ] 商品の特徴が一目で分かる
- [ ] ブランドカラーが統一されている
- [ ] 店頭で目立つデザインになっている
- [ ] 情報の優先順位が明確
```
