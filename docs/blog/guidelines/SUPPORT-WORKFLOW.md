## 5. 改善プロセス

### 5.1 フィードバック分析

**収集→分類→優先順位→対応策→実施→評価**

| ステップ | 頻度 | 担当 |
|----------|------|------|
| 収集 | 四半期ごと | 全チーム |
| 分析 | 四半期ごと | 編集部 |
| 優先順位 | 四半期ごと | チームリーダー |
| 対応策 | 四半期ごと | 担当チーム |
| 実施 | 月次 | 担当チーム |
| 評価 | 四半期ごと | 全チーム |

### 5.2 改優先度

| 優先度 | 基準 | 対応期間 |
|--------|------|----------|
| 高 | 多数の要望、重大な問題 | 1ヶ月以内 |
| 中 | 複数の要望、中程度の問題 | 3ヶ月以内 |
| 低 | 個別の要望、軽微な問題 | 6ヶ月以内 |

### 5.3 改善実績

改善内容は以下の方法で共有：

1. **Slackアナウンス**: 即時共有
2. **メール通知**: 詳細説明
3. **ドキュメント更新**: ガイドライン改訂
4. **トレーニング**: 必要に応じて実施

## 6. トラブルシューティング

### 6.1 よくある問題

**Q: ガイドラインが見つからない**
A: 以下のパスを確認してください
- メイン: `docs/blog/EXPERT_BLOG_WRITING_GUIDE.md`
- ガイドライン一覧: `docs/blog/guidelines/`

**Q: SEOチェックリストが長すぎる**
A: 重要項目から優先してください
- [ ] タイトルにメインキーワード
- [ ] メタディスクリプション120-160文字
- [ ] H2にキーワード配置
- [ ] 最初の段落にメインキーワード

**Q: nanobananaで画像がうまく生成されない**
A: 以下を試してください
1. プロンプトを簡潔に（200-400文字）
2. 英語を中心にする
3. 品質キーワードを追加

**Q: テンプレートが合わない**
A: 基本構造は維持しつつ、柔軟に対応可能です。メンターに相談してください。
