## 3. ロールバックトリガーの検証

### 3.1 ロールバック条件

| トリガー | 重大度 | 対応アクション |
|----------|--------|---------------|
| ライターフィードバックが3/5以下 | 高 | Phase 4から再実施 |
| SEOチェックリスト達成率<80% | 高 | SEO戦略を再検討 |
| ナビゲーション重大問題 | 中 | 即時修正 |
| 画像生成品質問題 | 中 | プロンプト改善 |

### 3.2 ロールバック手順

```mermaid
graph LR
    A[問題検知] --> B{重大度評価}
    B -->|高| C[即時停止]
    B -->|中| D[改善計画]
    B -->|低| E[次回更新へ]
    C --> F[原因分析]
    D --> F
    F --> G[修正実施]
    G --> H[再検証]
    H --> I[再開 or 継続改善]
```
