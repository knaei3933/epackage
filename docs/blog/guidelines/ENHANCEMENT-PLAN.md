# ブログガイドライン強化計画

> **目的**: Epackage Labブログ作成ガイドラインの段階的強化
> **策定日**: 2025-04-11
> **対象期間**: 2025年Q2-Q4（6ヶ月間）

---

## RALPLAN-DR サマリー

### 原則（Principles）

1. **専門性とSEOの両立**: パッケージ製造業界の専門知識を正確に伝えつつ、Google検索での上位表示を実現
2. **多言語対応の推進**: 日本語を基盤としつつ、韓国語市場への拡張を優先
3. **実用的な執筆支援**: ライターがすぐに使える具体的なテンプレートとプロンプトを提供
4. **継続的改善**: 四半期ごとのレビューと更新サイクルの確立

### 意思決定ドライバー（Decision Drivers）

| ドライバー | 優先度 | 影響 |
|-----------|--------|------|
| **市場拡大** | 高 | 韓国語対応で市場規模2倍 |
| **コンテンツ品質** | 高 | 専門性で信頼性構築 |
| **SEO効果** | 高 | 検索流入30%増加目標 |
| **執筆効率** | 中 | ライター教育コスト削減 |
| **技術的整合** | 中 | nanobanana MCP活用 |

### 実行可能オプション（Viable Options）

#### オプションA: 漸進的強化（推奨）
- **Q2**: 韓国語版ガイドライン基本版
- **Q3**: 記事テンプレート拡張、高度SEO戦略
- **Q4**: AI活用機能の統合

#### オプションB: 急速展開
- **Q2**: 全機能一括リリース
- **リスク**: 品質低下、教育負荷増大

#### オプションC: 最小限展開
- **Q2-Q4**: 既存機能の微調整のみ
- **リスク**: 競合他社に後れを取る

**採用**: オプションA（漸進的強化）


## nanobanana MCP統合計画

### 1. 現状と課題

#### 既存対応

- ✅ blog-image-nanobanana-guide.md (基本ガイド)
- ✅ 記事別プロンプト（17記事分）
- ✅ 画像サイズ規格

#### 課題

- ❌ プロンプト最適化プロセスが未確立
- ❌ 品質評価基準が不明確
- ❌ バージョン管理が不十分
- ❌ ライター教育リソースが不足

### 2. 統合強化計画

#### Phase 1: プロンプトライブラリ拡張（Q2）

```
shared/image-guidelines/
├── nanobanana-mcp.md (既存・拡張)
├── prompt-library/
│   ├── product-photography.md (新規)
│   ├── technical-diagrams.md (新規)
│   ├── lifestyle-images.md (新規)
│   └── infographics.md (新規)
├── quality-standards.md (新規)
└── optimization-tips.md (新規)
```

**内容**:
- 50パターン→100パターンへ拡張
- 用途別最適化ガイド
- 品質評価チェックリスト

#### Phase 2: ワークフロー統合（Q3）

**統合ポイント**:
1. 記事テンプレートへのプロンプト埋め込み
2. 自動生成スクリプトの開発
3. バージョン管理システムの導入

**ツール連携**:
- nanobanana MCP ←→ Markdownエディタ
- 生成画像の自動命名・配置
- Altテキスト自動生成

#### Phase 3: AI活用高度化（Q4）

**高度機能**:
- A/Bテスト対応（複数プロンプト比較）
- 品質スコアリング（AI評価）
- 自動最適化（フィードバック学習）

### 3. 教育プログラム

#### ライター向けトレーニング

| レベル | 内容 | 所要時間 |
|--------|------|----------|
| **初級** | 基本的なプロンプト作成 | 1時間 |
| **中級** | 用途別最適化 | 2時間 |
| **上級** | 品質評価・改善 | 3時間 |

#### 認定制度

- **ベーシック**: 3画像生成・評価合格
- **アドバンスト**: 10画像生成・品質スコア80%以上
- **エキスパート**: 20画像生成・指導能力あり
