# Epackage Lab Website Enhancement PRD

## プロジェクト概要

Epackage Labの日本市場向けウェブサイトの機能拡張とブランドイメージ向上プロジェクト。

## 目的

- ブランドアイデンティティの統一と向上
- ユーザーエクスペリエンスの最適化
- 見積システムの統合と機能強化
- コンテンツの日本語化とローカライゼーション
- ファイル構成の標準化

## 主要要件

### 1. ブランドカラーシステム
**要件**: 5EB6AC（緑色）と2f333d（紺色）をブランドの基本色として全ページに適用

**実装内容**:
- CSS変数の定義と適用
- 全コンポーネントでのカラーテーマ統一
- ブランドガイドラインの実装

### 2. ナビゲーションメニュー最適化
**要件**: ヘッダーの製品カタログドロップダウンメニューの日本語化と構成変更

**実装内容**:
- 「スタンディングパウチ」→「スタンドパウチ」に変更
- 「三方シール平袋」→「平袋」に変更
- 「チャック付き平袋」の位置調整
- 不要メニュー項目の削除

### 3. 言語ローカライゼーション
**要件**: 韓国語コンテンツの日本語翻訳

**実装内容**:
- 製造工程の5段階説明の日本語化
- サンプルリクエストの日本語化
- Epackage Labならではの強みセクションの日本語化
- コールトゥアクションセクションの日本語化

### 4. 見積システム統合
**要件**: 複数の見積機能を単一システムに統合

**実装内容**:
- quote-simulatorのデザイン適用
- roi-calculatorの機能統合
- サイズ設定のドラッグ＋入力機能
- 単価・追加料金表示の削除

### 5. 後加工プレビュー機能
**要件**: 見積システムでの後加工オプションの視覚的プレビュー

**実装内容**:
- 後加工選択時のプレビュー表示
- 実際の加工画像の活用
- 価格影響のリアルタイム計算
- カテゴリー別オプション整理

### 6. カタログダウンロード機能
**要件**: 会社概要ページでのPDFカタログダウンロード

**実装内容**:
- ダウンロードボタンの実装
- PDFファイルとの連携
- ダウンロードトラッキング

### 7. データテンプレート機能
**要件**: 注文データ入力テンプレートの提供

**実装内容**:
- テンプレートダウンロード機能
- 事前データ入力の誘導
- 入力補助システム

### 8. ファイル命名規則の標準化
**要件**: 韓国語ファイル名・フォルダ名の英語化

**実装内容**:
- フォルダ名の英語変換
- ファイル名の英語変換
- コード内参照の更新

### 9. コンテンツセクション削除
**要件**: 不要なメインページセクションの削除

**対象セクション**:
- 6種類の専門パウチ製品 完全カスタマイズ対応
- 30秒でわかる包装材費用 正確なお見積もりを即時計算
- 事業規模に合わせた最適なプラン 透明性の高い価格設定
- 工場見学・製造工程のご案内

### 10. 製造技術セクション強化
**要件**: 最新技術による高品質製造セクションの改善

**実装内容**:
- 信頼性向上のための画像活用
- 一貫サービスの強調
- 印刷・ラミネート・加工の3点写真表示

## 技術仕様

### フロントエンド
- Next.js 16 with App Router
- React 19 + TypeScript
- Tailwind CSS
- 日本語対応

### バックエンド
- Node.js
- SendGrid（メール機能）
- Supabase（データベース）

### 画像アセット
- 格納場所: `/images/`
- カテゴリー分類: post-processing, catalog, inbound-data
- WebP/AVIF対応

## 成功指標

- ユーザーエクスペリエンスの向上
- 見積コンバージョン率の改善
- ブランド認知度の向上
- 日本市場での受け入れ向上

- フェーズ2: 高度機能実装（1週間）
- フェーズ3: テストと最適化（1週間）
